RAG（Retrieval-Augmented Generation，檢索增強生成）的核心理念是：**讓大型語言模型（LLM）在回答問題前，先去「翻書」找資料。**

這解決了 LLM 容易產生「幻覺」（Hallucination）以及知識無法即時更新的問題。以下是 RAG 的五大核心概念：

---

### 1. 資料切分與向量化 (Indexing: Chunking & Embedding)
這是 RAG 的基礎工程。因為模型處理長度有限，我們必須先將資料處理成機器讀得懂的格式：
* **切分 (Chunking)：** 將長文章或代碼庫拆解成小的區塊（Chunks）。
* **向量化 (Embedding)：** 使用 Embedding 模型將這些文字區塊轉換成一串數字（向量）。這能捕捉文字間的**語義關係**，而不僅僅是字面匹配。
* **向量數據庫 (Vector DB)：** 將這些數字存存在專門的資料庫中（如 Pinecone, Milvus, Qdrant），方便後續快速比對。

### 2. 檢索 (Retrieval)
當使用者提出問題時，系統會進行搜尋：
* 將使用者的問題也轉換成**向量**。
* 在向量數據庫中尋找與問題「距離最接近」（語義最相關）的前 $k$ 個資料區塊。
* **關鍵技術：** 餘弦相似度（Cosine Similarity）是判斷相關性最常用的數學指標。

### 3. 增強 (Augmentation)
這是一個「封裝」的過程。系統不會直接把問題丟給模型，而是重新組合 Prompt：
* **模板：** `「請根據以下參考資料回答問題：{檢索到的內容}。問題是：{使用者原始問題}」`
* 這樣做的目的是將模型限制在給定的知識範圍內，減少它「瞎掰」的機率。

### 4. 生成 (Generation)
最後，LLM（如 Claude 3.5 或 GPT-4）讀取了包含背景資料的 Prompt，產生回答。
* 因為有了「教科書」（檢索到的資料），模型可以給出更精確、帶有引用來源的答案。

### 5. 評估與優化 (RAG Triad)
一個好的 RAG 系統通常會根據以下三個維度進行優化：
* **脈絡相關性 (Context Relevance)：** 檢索出來的資料真的跟問題有關嗎？
* **回答忠實度 (Faithfulness)：** 模型的回答是否有根據檢索到的資料（而非自己亂編）？
* **答案相關性 (Answer Relevance)：** 回答是否真的解決了使用者的問題？

---

### 為什麼 RAG 對開發者（如後端工程師）很重要？

| 優點 | 說明 |
| :--- | :--- |
| **低成本** | 不需要花大錢重新訓練（Fine-tune）模型，只需更新資料庫。 |
| **即時性** | 資料庫一更新，AI 就能立刻存取最新資訊（如最新的 API 文件）。 |
| **可解釋性** | AI 可以標註答案來源（例如：根據文件第 5 頁），增加信任度。 |
| **隱私控管** | 可以根據權限過濾檢索內容，確保模型不會讀到不該讀的敏感資料。 |
